# browser-automation-cli

> CLI one-shot de automação Chrome CDP para agentes de IA, sem daemon e sem telemetria.

browser-automation-cli é uma CLI Rust de processo único que lança Chrome ou Chromium do sistema via CDP, executa ações de browser do agente e depois finaliza e sai. Ciclo de vida: BORN, EXECUTE, FINALIZE, DIE. Settings de produto usam flags da CLI e config XDG apenas (sem env vars de produto).

## Superfície de Comandos (v0.1.1)
- Descoberta: doctor, commands, schema, version, completions
- Config XDG: config init|path|show|get|set
- Navegação: goto, back, forward, reload
- Scrape local: scrape (--format/--engine), batch-scrape, crawl, map, search, parse
- Interação: view, press, write, type, keys, wait (multi --text OR), hover, drag, fill-form, upload, click-at
- Observação: extract, text, scroll, attr, assert, grab --path
- Abas: page, dialog, cookie
- Captura: console, net
- MITM: mitm init-ca|start|status|list|get|har|export|domains|apis
- Workflow: workflow run|resume|status
- Avançado: eval, emulate, resize, perf, lighthouse, screencast, heap, extension, devtools3p, webmcp, run, exec

## Contrato de Settings
- Settings de produto: flags da CLI + XDG `config.toml` apenas
- Sem env vars de produto (`BROWSER_AUTOMATION_CLI_*` não devem ser inventadas)
- Vars de SO fora de settings de produto: `RUST_LOG`, `NO_COLOR`
