# browser-automation-cli

> CLI one-shot de automação Chrome CDP para agentes de IA, sem daemon e sem telemetria remota.

browser-automation-cli é uma CLI Rust de processo único que lança Chrome ou Chromium do sistema via CDP, executa ações de browser do agente e depois finaliza e sai. Trabalho multi-passo deve usar `run --script` dentro de um processo para que refs `@eN` permaneçam válidas. Ciclo de vida: BORN, EXECUTE, FINALIZE, DIE.

Settings de produto são comandos, flags e XDG via `config path|init|show|set|get|list-keys` apenas. Logging usa `--verbose`/`--debug`/`-q` ou `config set log_level`. Cor usa `config set color true|false`. Chrome e Lighthouse usam path do shell ou `config set chrome_path|lighthouse_path`.

## Contrato de Settings
- Settings de produto: comandos, flags, XDG via `config path|init|show|set|get|list-keys` apenas
- Encryption: `config set encryption_key` apenas
- Logging: `--verbose`/`--debug`/`-q` ou `config set log_level` / `log_to_file`
- Cor: `config set color true|false`
- Chrome / Lighthouse: path do shell ou `config set chrome_path` / `config set lighthouse_path` (mais flag lighthouse)
- Cache Redis: `config set cache_backend redis` + `cache_redis_url` (somente `redis://`; `rediss://` fail-closed); doctor `cache_redis`

## Bases de Contrato
- Envelope de sucesso: `{"schema_version":1,"ok":true,"data":...}`
- Envelope de erro com `--json`: `{"schema_version":1,"ok":false,"error":{...}}` (fail-fast multi-passo pode incluir `data.steps` parciais)
- Exit codes incluem 0, 2, 65, 66, 69, 70, 74, 78, 124, 130, 141, 255
- Fragmentos de schema ao vivo: `browser-automation-cli schema --cmd <name> --json`
